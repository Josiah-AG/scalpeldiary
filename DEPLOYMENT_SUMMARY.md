## 🔐 Security Notes

### Required Secrets

You'll need to generate/save these:

1. **JWT_SECRET** - Generate 32+ random characters
   ```bash
   # Generate on Mac/Linux
   openssl rand -base64 32
   ```

2. **AWS Credentials** - From AWS IAM
   - AWS_ACCESS_KEY_ID
   - AWS_SECRET_ACCESS_KEY

3. **Database URL** - Auto-generated by Railway

### Important Security Steps

- [ ] Change JWT_SECRET from default
- [ ] Change master password after first login
- [ ] Remove development URLs from CORS
- [ ] Enable HTTPS only (automatic with Cloudflare)
- [ ] Set up Cloudflare WAF rules
- [ ] Regular database backups

## 🆘 Support Resources

### If Something Goes Wrong

1. **Check Logs**:
   - Railway: Dashboard → Service → Deployments → Logs
   - Cloudflare: Dashboard → Pages → Deployments
   - Browser: Developer Console (F12)

2. **Common Issues**:
   - CORS errors → Check backend CORS config
   - API not connecting → Verify VITE_API_URL
   - Database errors → Check migrations ran
   - S3 upload fails → Verify AWS credentials

3. **Documentation**:
   - DEPLOYMENT_GUIDE.md - Troubleshooting section
   - Railway docs: https://docs.railway.app
   - Cloudflare docs: https://developers.cloudflare.com
